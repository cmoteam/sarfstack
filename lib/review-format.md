# Standard Review Format — レビュー出力フォーマット

すべてのレビュー系エージェントは、以下の構造に従って出力してください。

## Required Sections

### 1. Summary（総括）
レビュー結果を1-2文で要約。最も重要な指摘を含む。

### 2. Verdict（判定）
レビューの種類に応じた判定:

| Agent | 判定 |
|-------|------|
| CEO Review | GO / PIVOT / KILL |
| CMO Review | A / B / C / D （総合評価） |
| Creative Director | APPROVE / REVISE / REWORK |
| SEO | A / B / C / D / F |
| UI Review | A / B / C / D / F |

### 3. Scorecard（スコアカード）
各評価軸のスコアをテーブルで表示。

### 4. Findings（発見事項）
優先度順に分類:
- **Critical（必須修正）**: これを直さないとリリースできない
- **Important（推奨修正）**: 直した方が明らかに良くなる
- **Nice to Have（余裕があれば）**: 改善の余地がある

### 5. Specific Fixes（具体的な修正案）
問題指摘には必ず修正案を添える。
「ここがダメ」だけでなく「こう変えるべき」まで書く。

原文と修正案をペアで提示:
```
- 原文: 「...」
- 修正案: 「...」
- 理由: ...
```

### 6. Next Steps（次のアクション）
チェックリスト形式で、具体的なアクションを記述。
可能であれば担当エージェントも指定。

## Formatting Rules

- テーブルを積極的に使う（可読性向上）
- 箇条書きは3-5項目に収める（多すぎると読まれない）
- 具体的な数字・データを添える
- スコアだけでなく、なぜそのスコアかを1文で添える
