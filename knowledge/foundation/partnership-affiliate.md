# Partnership & Affiliate — パートナーシップとアフィリエイト

「既に顧客を持っている人と組む」ことが、最短・最安の成長戦略になり得る。
広告や検索が飽和し単価高騰する一方、**パートナーシップは複利で効く・関係が資産になる**。
ただし**合意設計・経済設計・関係の継続**が難しく、片手間では成果が出ない領域でもある。

## 4つのパートナー種別

| 種別 | 関係 | 経済モデル | 例 |
|------|------|-----------|------|
| **Affiliate（アフィリエイト）** | 1対多 / 成果報酬 | CPA / Rev Share | ブロガー紹介、ポイントサイト |
| **Co-Marketing（共同マーケ）** | 1対1 / 対等 | 相互送客 | 共同ウェビナー、共同レポート |
| **Channel / Reseller（販売代理店）** | 1対1 / 縦 | 卸売/Rev Share | 代理店網、システムインテグレータ |
| **Strategic Alliance（戦略提携）** | 1対1 / 深い | 多様（投資・共同開発含む） | 技術統合、OEM、Joint Venture |

## Co-Marketing（共同マーケティング）

**補完的な顧客ベースを持つ2社が、相互にリーチを交換する**。

### 成功する組合せの条件
1. **ICPが重複するが競合しない** — 同じ顧客に別の課題を解決
2. **ブランド格が近い** — 片方が"上"だと関係が歪む
3. **実行スピードが近い** — 片方が遅いと疲弊
4. **経営層のコミット**がある

### 代表的な型

| 型 | 内容 | 想定CV経路 |
|----|------|----------|
| **共同ウェビナー** | 両社専門家が登壇、両社リストに告知 | 参加者リスト交換 |
| **共同ホワイトペーパー / レポート** | 共著調査レポート、両社LPで配布 | DL者リード共有 |
| **共同ローンチ / キャンペーン** | 同時発表・相互LP設置 | 相互送客 |
| **API / Integration連携** | 製品統合＋共同マーケ | プロダクト内接点 |
| **ポッドキャスト / Newsletter相互出演** | 相手オーディエンスへの露出 | 認知＋間接的誘導 |
| **共同イベント / Meetup** | リアル集客をシェア | 名刺交換・質的接点 |

### 成功の実務ルール
- **Role & Responsibility を文書化** — 誰が何を書き、いつまでに、誰がレビューするか
- **KPIを事前合意** — リード数 / MQL / 共同CV
- **リード共有ポリシー** — オプトイン / 排他期間 / データクリーンルーム
- **ブランドガイドライン交換** — ロゴ使用規定・NG表現
- **振り返りミーティング** — やりっ放しにしない

## アフィリエイト設計

### 基本フロー

```
Affiliate（発信者） → Unique Link / Coupon → Landing Page
                                                  ↓
                                              Conversion
                                                  ↓
                                              Tracking
                                                  ↓
                                              Commission Payout
```

### 報酬設計

| モデル | 内容 | 向いているケース |
|--------|------|-----------------|
| **CPA（固定）** | 1件CV = ¥X | シンプル、予算管理しやすい |
| **Rev Share（売上%）** | 売上の X% | SaaS、継続課金商材 |
| **Recurring（継続）** | 契約継続中ずっと % | SaaS定番、12〜24ヶ月限定も |
| **Tiered** | 成果ボリュームで料率UP | トップアフィリエイト育成 |
| **Hybrid** | 固定＋成果 | 大型パートナー |

### SaaS標準レンジ
- 初回課金の **20〜30%（Recurring 12ヶ月）**
- または **初回ACVの20〜50%（一括）**

### 商品別の典型的料率

| カテゴリ | 典型料率 |
|---------|--------|
| SaaS | 10〜30% Recurring |
| EC（一般商材） | 3〜10% |
| 金融商品 | CPA ¥5K〜¥30K |
| 教育・情報商材 | 30〜50% |
| 暗号資産 | 10〜30% |

### ASP（アフィリエイトサービスプロバイダー）
日本: A8.net、バリューコマース、afb、アクセストレード、もしもアフィリエイト
グローバル: Impact.com、PartnerStack、ShareASale、Rewardful、Tapfiliate、CJ Affiliate

**SaaSなら PartnerStack / Rewardful が使いやすい。**

## アフィリエイトの品質管理

### 警戒すべきパターン
- **ブランドキーワード入札** — 自社指名検索でパートナーが広告出稿 → 自社広告費と食い合い
- **Cookie Stuffing** — ユーザーの同意なく勝手にトラッキング
- **誤解を招く比較記事** — 自社/競合を意図的に貶める
- **クーポンサイト偏重** — 本来Organicで購入する層の横取り（Incrementality低い）

### 対策
- **Terms & Conditions** で禁止行為を明文化
- ブランド入札禁止、SEOでのNG戦術を明記
- 月次パフォーマンスレビュー＆違反者の排除
- **Incrementality Test**で真の貢献度を測る

## チャネル / リセラー / SI連携

### Enterprise B2Bで強い
- 地域/業界別の深い関係を代理店が持つ
- プロダクトだけでは届かない層にリーチ
- インプリ・サポートを代理店に委譲

### 代表的モデル
| モデル | 内容 |
|--------|------|
| **Reseller（再販）** | 自社プロダクトを代理店が自社契約として販売 |
| **Referral Partner** | 紹介のみ、契約は本体と |
| **Value-Added Reseller（VAR）** | 再販＋独自のサービス/カスタマイズ |
| **System Integrator（SI）** | 導入・実装を担う |
| **MSP（Managed Service Provider）** | 運用も継続受託 |

### 経済設計
- **Margin（卸売差）** 15〜40%
- **Deal Registration** — 代理店が商談登録することで重複を防ぐ
- **Certification Program** — 認定レベル（Gold / Silver）で特典差別化
- **Marketing Development Funds（MDF）** — 共同マーケ用予算

## 紹介プログラム（Referral Program）

**既存顧客からの紹介**は最強のCAC削減施策。LTVも高い。

### 設計の型

| 型 | 内容 | 例 |
|----|------|-----|
| **Dual-sided** | 紹介者と被紹介者両方に特典 | Dropbox（両者500MB追加） |
| **紹介者のみ** | 紹介者に報酬 | 多くのSaaS |
| **被紹介者のみ** | 新規割引 | EC系 |
| **チャリティ型** | 紹介額を寄付 | 社会的ブランド |

### Dropbox事例
- 招待者・被招待者ともに500MB追加
- 8ヶ月で新規登録4倍、2.8Mユーザー獲得
- **相互的報酬設計**が紹介率を最大化

### 実装ポイント
- **紹介の動機づけが先、リワードは二次** — 良い製品体験があって初めて紹介が起きる
- **紹介しやすいUI** — 1クリックでリンク共有、SNS連携
- **進捗の可視化** — 「あと1人で特典」など残りが見える
- **タイミング** — Aha Moment直後がベスト
- **追跡** — Unique Link / Email Attribution

### 紹介率の目安
- B2C SaaS: 10〜20%
- B2B SaaS: 20〜40%（Friendbuy)
- D2C: 5〜15%

## エンベデッドパートナーシップ（API / Integration）

**プロダクト同士が統合される**時点で、両社のユーザーは自然に露出する。

### 典型パターン
- Slack × Zapier × SaaS: Slackの仕事場にSaaSの通知が入る
- Shopify × Meta: Shopify EC所有者がMeta広告簡単出稿
- Stripe × SaaS: 決済導入時にSaaSを勧める

### 成功要因
1. **両社のユーザーにとって便益がある**（片方だけ得する統合は続かない）
2. **マーケットプレイス掲載**（Shopify App Store / Salesforce AppExchange / Slack Directory）
3. **共同GTM施策**（ローンチ告知、共同ケーススタディ）
4. **プレミアム配置**（マーケットプレイスTop表示は最大流入源）

## パートナーシップKPI

| KPI | 意味 |
|-----|------|
| **Partner-Sourced Revenue** | パートナー起点の売上 |
| **Partner-Influenced Revenue** | パートナーが関与した売上（重複考慮） |
| **Partner CAC** | パートナー経由獲得コスト |
| **Partner LTV / LTV:CAC** | 他チャネル比較用 |
| **Activation Rate** | パートナー契約 → 実際に送客してくる率 |
| **Top-20% Partners Concentration** | トップ20%のパートナーが売上の何%か |

**パートナーは80:20則が強く出る** — 上位20%で売上の80%以上。

## よくある失敗

| 罠 | 実態 |
|------|------|
| 「全員にメール送って説明会」 | 10社深く > 100社浅く |
| 「契約だけして放置」 | Activation率が低く、売上に繋がらない |
| 「料率だけ高く設定」 | 関係性・トレーニング・素材提供が伴わないと動かない |
| 「PRMツール入れれば動く」 | ツールは補助、人的関係が本体 |
| 「本体マーケと分断」 | パートナー経由のLPやコピーが統一されていない |
| 「計測が粗い」 | Incrementalityが測れず予算正当化できない |

## SAAFとの接続

- **Set**: 自社のICP・補完的サービス・潜在パートナー候補を `memory/company/` に書く
- **Ask**: 「この商材に最適なCo-Marketingパートナー候補を10社」「Referralプログラム設計を3案」
- **Action**: 契約書締結、共同ウェビナー開催、アフィリエイトASP導入、Referral UI実装
- **Feedback**: Partner-Sourced Revenue、LTV、Activation率を `memory/results/performance-data.md` に記録
